Generate Backup of the SQLite Database

Your stock portfolio database contains valuable investment information such as purchases, sales, dividends, and historical market prices. Protecting this data is essential for maintaining a reliable investment record.

Creating periodic backups of the SQLite database ensures that your portfolio information can be restored if your computer experiences hardware failure, accidental deletion, or other unexpected problems.

Why Database Backups Are Important

The SQLite database stores all portfolio transactions and historical records used by the software to calculate performance metrics and investment returns.

Without backups, recovering lost data may be impossible. Regular backups help safeguard years of investment history and analysis.

How the Backup Process Works

The software provides a function that creates a copy of the SQLite database file containing your portfolio data.

This backup file can be stored in another folder, external drive, or cloud storage location to provide additional protection.

Recommended Backup Practices

To ensure your investment data remains safe, consider following these backup practices:

  • Create backups regularly
  • Store backups on a separate device
  • Maintain multiple backup versions
  • Use cloud storage or external drives

These practices reduce the risk of permanent data loss.

Restoring the Database from a Backup

If necessary, the backup file can be used to restore your portfolio database. This process replaces the current database file with the backup copy.

Once restored, the software will immediately recognize all previously recorded transactions and historical data.

Protecting Years of Investment Data

Many investors track their portfolios for years or even decades. A reliable backup strategy ensures that this valuable financial history remains protected.

By generating periodic SQLite backups, you can confidently manage your portfolio knowing that your data remains safe.
